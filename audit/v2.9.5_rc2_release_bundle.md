# v2.9.5 rc2 Release Audit Bundle

Date: 2026-04-06
Status: assembled rc2 release-hardening bundle

This memo is the bundle view for rc2 release judgement.
It collects the design, operations, and verification material that was previously spread across
stacked rc2 branches.

## What rc2 changed

rc2 changed shell, validation, and operator-facing contract hardening only:

- role-safe `crisp-v29` facade with banner, summary, and config-role fail-fast
- managed theta provenance / stale-use governance with `theta_rule1_resolution.json`
- Cap truth-source reconciliation audit wired into replay / validators
- Rule3 trace diagnostics sidecar with ordering and handling summaries
- stale artifact / misuse guards for manifest, theta, role contamination, and local heavy-run warnings
- writer / validator / report responsibility split on the shell side
- sidecar policy, CI boundary, local heavy-run boundary, and operator help unification

## What rc2 did not change

rc2 did not change frozen object logic or release taxonomy semantics:

- Core object logic and SCV semantics remain frozen
- Rule3 remains trace-only and observational
- benchmark remains the only same-config regression baseline
- production remains cross-regime and is not redefined as a benchmark equivalent
- `cap_batch_eval.json` remains the Cap truth source
- `run_manifest.json` and `output_inventory.json` stay under the frozen top-level schema policy
