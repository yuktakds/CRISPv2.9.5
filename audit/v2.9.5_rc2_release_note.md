# v2.9.5 rc2 Release Note

Date: 2026-04-06
Release anchor: `v2.9.5-rc2`
Anchor commit: `5773a6fb51f8cca6ef065c07368c24b92776aaa7`

This is the one-page release note for rc2 judgement.
If you need the full supporting record, start from [v2.9.5_rc2_release_bundle.md](v2.9.5_rc2_release_bundle.md).

## What changed

- added the role-safe `crisp-v29` facade with fixed banner, summary, and config-role fail-fast
- hardened managed theta governance and recorded provenance in `theta_rule1_resolution.json`
- added Cap truth-source reconciliation checks to replay and validators
- added Rule3 trace diagnostics in `rule3_trace_summary.json` without changing semantics
- added stale artifact / misuse guards for manifest, theta, truth-source, and role contamination
- separated writer / validator / report responsibilities in the v29 shell
- fixed the rc2 operator boundary in docs: required CI, local heavy-run checklist, sidecar policy, and help text

## What did not change

- Core object logic and SCV semantics did not change
- Rule3 is still trace-only and observational
- benchmark is still the only same-config regression baseline
- production is still cross-regime and is not a regression baseline
- `cap_batch_eval.json` is still the Cap truth source
- `run_manifest.json` and `output_inventory.json` stayed inside the frozen schema policy
